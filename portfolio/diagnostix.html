DIAGNOSTIX

A documented exploration of Linux-based system diagnostics, tooling, and troubleshooting workflows.

Project Overview

DiagnostiX is a custom Linux-based diagnostic environment created as a hands-on exploration of system analysis, troubleshooting, and tooling integration. The goal was not to produce a commercial product, but to deepen understanding of how diagnostic tools interact with hardware, operating systems, and real-world failure scenarios.

This page documents DiagnostiX in a stable, working state. Active development is currently paused while I focus on other priorities, but the project remains viable and is planned to continue when time and focus allow.

What This Project Explored

  • Linux system inspection and hardware identification
  • Disk, memory, and process diagnostics
  • Command-line tooling and workflow design
  • Bootable environments and recovery use cases
  • Practical troubleshooting methodology

Context

DiagnostiX represents a point in my technical journey where I was focused on understanding systems from the inside out. It sits alongside my home lab work, website development, and infrastructure experiments as part of a broader systems-first learning approach.
